# 回测方法论（P4）

## 1. 目标

本文档定义策略信号与回测模块的统一口径，确保收益评估可重复、可审计、可解释。

## 2. 核心原则

1. 禁止未来函数：任一交易日 `t` 的决策只使用 `t` 及之前的数据。
2. 固定快照复现：同一输入快照与参数必须得到同一组 KPI。
3. 成本显式建模：手续费与滑点在每次仓位变更时计入权益。
4. 风险优先：除收益外，必须同时输出回撤与波动相关指标。

## 3. 数据口径

1. 基础价格序列使用基金历史净值 `unit_nav`。
2. 场外基金（如 `014943`）不直接使用盘口语义，资金流因子可通过关联 ETF（如 `159870`）代理。
3. 回测期间使用一次性历史快照并计算 `snapshot_hash`。

## 4. 信号与成交假设

1. 在交易日 `t` 收盘后生成信号。
2. 目标仓位在 `t -> t+1` 收益区间生效。
3. 仓位变更成本：
   - `trade_cost = equity * |Δposition| * (fee_rate + slippage_bps / 10000)`

## 5. KPI 指标

至少输出以下指标：

1. `total_return_pct`
2. `annualized_return_pct`
3. `max_drawdown_pct`
4. `win_rate`
5. `sharpe`
6. `turnover_ratio`
7. `trade_count`

## 6. 可复现要求

1. 回测结果需记录：`snapshot_hash`、`strategy_params`、`backtest_config`、`weight_version`。
2. 相同输入重复运行时，策略 KPI 与交易序列应一致。
3. 若结果不一致，视为实现缺陷，需要阻断发布。

## 7. 已知限制

1. 当前模型未纳入申赎费、冲击成本与流动性容量约束。
2. 情绪因子为关键词法，后续可替换为更强语义模型。
3. 当前日频回测不覆盖盘中交易执行细节。
